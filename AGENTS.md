# ChefSpark Agent Instructions

Read these documents before making changes:

- `RULES.md`
- `PROJECT_CONTEXT.md`
- `PROJECT_HANDOVER.md`

Always follow `RULES.md`.

## Working Principles

- Treat ChefSpark as a real mobile-first product, not only a university MVP.
- Preserve the existing React, Vite, Express, and SQLite architecture.
- Do not introduce technologies outside the approved stack without explicit approval.
- Prefer incremental improvements over broad rewrites.
- Explain major architectural decisions before implementation.
- Keep the interface mobile-first, simple, fast, and polished.
- When proposing an AI feature, explain the workflow advantage it offers over a generic AI chatbot.

## Workspace Rules

- The repository root is the folder containing this `AGENTS.md` file.
- Do not create or modify files outside the repository root.
- Explain why a new file is needed before creating it.
- Do not create duplicate documents or files with overlapping purposes.
- Do not modify unrelated files.

## Document Roles

- `README.md`: user-facing product overview and setup instructions.
- `PROJECT_CONTEXT.md`: concise current product and technical context.
- `PROJECT_HANDOVER.md`: detailed product history, status, decisions, and future direction.
- `RULES.md`: development and collaboration constraints.
- `SMARTKITCHEN_Workflow_v1.md` and `SMARTKITCHEN_Technical_Design_v1.md`: historical MVP documents; do not treat them as current requirements.
